# بطاقة نموذج وقفة التجريبي

## الهوية والنطاق

- الإصدار: `0.2.0-hackathon`
- تاريخ القياس: 2026-07-16
- الغرض: تنبيه وقائي قبل قرار تقسيط جديد.
- خارج النطاق: التقييم الائتماني، قبول أو رفض التمويل، أو ادعاء احتمال تعثر مصرفي.

## مكونات النظام

1. `DailyCashFlowEngine`: محاكاة حتمية يومية لمدة 90 يومًا.
2. `TemporalRiskEngine`: مؤشر ضغط قابل للتفسير؛ نقاط كل عامل تجمع فعليًا إلى الدرجة.
3. `FinancialDecisionEngine`: مقارنة قبل/بعد، بدائل، واختبارات صدمة.
4. `BehavioralLearningService`: متعلم UCB1 محلي يوازن بين ثلاثة تدخلات سلوكية.
5. Groq LLM/Vision: صياغة شرح عربي وقراءة الفاتورة بعد موافقة المستخدم؛ لا يحسب المؤشر المالي.

## بيانات التحقق

- المصدر: بيانات اصطناعية مولدة برمجيًا، وليست بيانات عملاء.
- العدد: 100 سيناريو، 20 لكل نمط.
- الأنماط: مستقر، تراكم BNPL، أزمة توقيت، دخل متذبذب، هش للطوارئ.
- seed: `20260716`.
- الحقيقة المرجعية: عجز يومي، التزام إلزامي غير مغطى، أو رصيد سالب مستمر وفق المحاكاة.
- عتبة التنبيه: 50/100.

## القيود المعروفة

- النتائج تصف العينة الاصطناعية المحددة فقط ولا تمثل دقة على عملاء حقيقيين.
- معدل الإنذار الزائد يحتاج معايرة ببيانات Pilot قبل الإنتاج.
- التواريخ والدخل المتذبذب والصدمات في النموذج الاصطناعي تبسيط للواقع.
- UCB1 يحتاج تفاعلات كافية قبل أن يصبح تفضيله ذا معنى؛ يبدأ بالاستكشاف.
- SharedPreferences مناسب للنموذج الأولي فقط، ويجب استبداله بتخزين مشفر في الإنتاج.
- مفاتيح خدمات الذكاء الاصطناعي يجب أن تنتقل إلى backend proxy قبل النشر العام.

## حواجز الأمان المقترحة للإنتاج

- موافقة صريحة وقابلة للسحب وتقليل البيانات.
- تشفير أثناء النقل وفي التخزين، وفصل هوية العميل عن خصائص النموذج.
- مراقبة الانحياز حسب الشرائح، والانحراف، والإنذارات الزائدة.
- fallback حتمي، سجل إصدار للنموذج، وإمكانية تعطيل التدخل.
- Pilot مع مجموعة مقارنة قبل أي ادعاء بخفض التأخر أو التعثر.
