# بطاقة نموذج وقفة التجريبي

## الهوية والنطاق

- الإصدار: `1.1.0+3`
- تاريخ القياس: 2026-07-16
- الغرض: تنبيه وقائي قبل قرار تقسيط جديد.
- خارج النطاق: التقييم الائتماني، قبول أو رفض التمويل، أو ادعاء احتمال تعثر مصرفي.

## مكونات النظام

1. `FinancialDecisionEngine`: محرك المنتج الفعلي؛ يحسب الدرجة، والمقارنة قبل/بعد، والبدائل، واختبارات الصدمة.
2. `DailyCashFlowEngine`: محاكاة حتمية يومية مستقلة لمدة 90 يومًا، تُستخدم كحقيقة مرجعية في الاختبار.
3. `BehavioralLearningService`: متعلم UCB1 محلي يوازن بين ثلاثة تدخلات سلوكية.
4. Groq LLM/Vision: صياغة شرح عربي وقراءة الفاتورة بعد موافقة المستخدم؛ لا يحسب المؤشر المالي.
5. `FinancialPreventionEngine`: 72 مسارًا احتماليًا بseed ثابت، ونافذة سقوط معرفة بثلاثة أيام متتالية تحت احتياطي الأمان، وأقل تدخل وزمن تعافٍ.
6. `GoalPlanEngine`: مساهمة واحتياطي وحد صرف وخطة أسبوعية حتمية؛ LLM يعيد الصياغة فقط.

## سيناريو العرض القابل لإعادة التشغيل

- الملف: `tool/demo_scenario.dart`، والنتيجة المثبتة: `docs/demo_scenario_output.json`.
- سارة الاصطناعية: 8,000 / 3,500 / 1,500 / 1,800، وقسط جديد 640.
- مؤشر القرار 54→86، احتمال نافذة الضغط 76%، اليوم 23.
- أقل تدخل: قسط 160 + حماية 300 من الصرف المتغير؛ المؤشر 54 والاحتمال 6%، مع 5 أيام هشّة متجنبة.
- هذه مخرجات محاكاة نموذج أولي، لا أثر عميل ولا احتمال تعثر مصرفي.

## بيانات التحقق

- المصدر: بيانات اصطناعية مولدة برمجيًا، وليست بيانات عملاء.
- العدد: 100 سيناريو، 20 لكل نمط.
- الأنماط: مستقر، تراكم BNPL، أزمة توقيت، دخل متذبذب، هش للطوارئ.
- seed: `20260716`.
- الحقيقة المرجعية: عجز يومي، التزام إلزامي غير مغطى، أو رصيد سالب مستمر وفق المحاكاة.
- محرك التقييم: نفس `FinancialDecisionEngine` المستخدم في شاشات التطبيق.
- عتبة التنبيه: 45/100، وهي نفس بداية مستوى «يحتاج وقفة» في المنتج.

## القيود المعروفة

- النتائج تصف العينة الاصطناعية المحددة فقط ولا تمثل دقة على عملاء حقيقيين.
- المحرك يفضّل عدم تفويت الخطر في هذه العينة، مقابل معدل إيجابيات كاذبة مرتفع يحتاج معايرة ببيانات Pilot قبل الإنتاج.
- التواريخ والدخل المتذبذب والصدمات في النموذج الاصطناعي تبسيط للواقع.
- UCB1 يحتاج تفاعلات كافية قبل أن يصبح تفضيله ذا معنى؛ يبدأ بالاستكشاف.
- SharedPreferences مناسب للنموذج الأولي فقط، ويجب استبداله بتخزين مشفر في الإنتاج.
- مفاتيح خدمات الذكاء الاصطناعي يجب أن تنتقل إلى backend proxy قبل النشر العام.

## حواجز الأمان المقترحة للإنتاج

- موافقة صريحة وقابلة للسحب وتقليل البيانات.
- تشفير أثناء النقل وفي التخزين، وفصل هوية العميل عن خصائص النموذج.
- مراقبة الانحياز حسب الشرائح، والانحراف، والإنذارات الزائدة.
- fallback حتمي، سجل إصدار للنموذج، وإمكانية تعطيل التدخل.
- Pilot مع مجموعة مقارنة قبل أي ادعاء بخفض التأخر أو التعثر.
